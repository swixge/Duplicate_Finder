# Duplicate Finder Application

Додаток для пошуку дублікатів у PDF та Word документах з використанням п'яти різних алгоритмів.

> 📚 **Повна документація:** Див. [DOCUMENTATION.md](DOCUMENTATION.md)

## Структура проекту (MVC)

```
App/
├── models/                      # Моделі (бізнес-логіка)
│   ├── __init__.py
│   ├── file_processor.py       # Обробка файлів
│   ├── search_algorithms.py    # Алгоритми пошуку
│   ├── algorithm_comparator.py # Порівняння алгоритмів
│   └── database_manager.py     # Управління БД
├── controllers/                 # Контролери (логіка додатку)
│   ├── __init__.py
│   └── main_controller.py      # Головний контролер
├── views/                       # Представлення (UI)
│   ├── __init__.py
│   ├── login_view.py           # Екран входу
│   └── main_view.py            # Головне вікно
├── main.py                      # Точка входу
├── requirements.txt             # Залежності
├── README.md                    # Цей файл
└── duplicates.db               # База даних (створюється автоматично)
```

## Встановлення

1. Встановіть залежності:
```bash
pip install -r requirements.txt
```

2. Запустіть додаток:
```bash
python main.py
```

## Особливості

### П'ять алгоритмів пошуку дублікатів:

1. **Rabin-Karp** - Швидкий алгоритм на основі хешування
2. **Suffix Tree** - Алгоритм на основі суфіксних дерев
3. **Semantic** - Семантичний пошук на основі синонімів
4. **Fuzzy Match** - Нечіткий пошук подібних текстів
5. **N-gram Analysis** - Аналіз повторюваних послідовностей слів

### Ролі користувачів:

- **Гість** - Базовий доступ до аналізу файлів
- **Адміністратор** - Доступ до історії пошуків та управління (пароль: admin123)

### Функціональність:

- Завантаження PDF та DOCX файлів
- Аналіз тексту (кількість символів, слів, унікальних слів тощо)
- Порівняння п'яти алгоритмів пошуку дублікатів
- Підсвітка найчастіших фраз у тексті (5 різних кольорів)
- Збереження результатів у базі даних
- Перегляд історії пошуків (для адміністраторів)
- Генерація HTML-файлів з підсвіткою
- Вибір окремих алгоритмів для аналізу

## Архітектура MVC

### Models (models/)
Містять бізнес-логіку:
- `FileProcessor` - Обробка та аналіз файлів
- `RabinKarpSearch`, `SuffixTreeSearch`, `SemanticSearch`, `FuzzyMatchSearch`, `NGramAnalysis` - 5 алгоритмів пошуку
- `AlgorithmComparator` - Порівняння результатів
- `DatabaseManager` - Управління SQLite БД

### Controllers (controllers/)
Координують взаємодію між моделями та представленнями:
- `MainController` - Управління потоком додатку

### Views (views/)
Відповідають за користувацький інтерфейс:
- `LoginView` - Екран входу
- `MainView` - Головне вікно з таблицею результатів та текстовим полем

## Використання

1. Запустіть додаток
2. Виберіть роль (Гість або Адміністратор)
3. Завантажте PDF або DOCX файл
4. Встановіть довжину підрядка для пошуку
5. Натисніть "Аналізувати"
6. Переглядайте результати та підсвічені дублікати

## Залежності

- **PyPDF2** - Обробка PDF файлів
- **python-docx** - Обробка Word документів
- **nltk** - Обробка природної мови
