## 💻 Використання

### Вхід до системи

#### Гість
- Просто натисніть "Увійти"
- Базовий доступ до аналізу файлів
- Можете завантажувати файли та аналізувати

#### Адміністратор
- Виберіть "Адміністратор"
- Введіть пароль: `admin123`
- Додатково можете переглядати історію всіх пошуків

### Аналіз файлу

1. Натисніть "Завантажити файл"
2. Виберіть PDF або DOCX файл
3. Встановіть "Довжину підрядка" (за замовчуванням 5)
4. Натисніть "Аналізувати"
5. Переглядайте результати в таблиці та підсвічені дублікати в тексті

### Перегляд попередніх пошуків (тільки адміністратор)

1. Натисніть "Попередні пошуки"
2. Виберіть файл зі списку
3. Натисніть "Показати статистику"
4. Переглядайте результати та текст з підсвіткою

### Результати аналізу

Таблиця показує для кожного алгоритму:
- **Кількість повторів** - Скільки унікальних дублікатів знайдено
- **Унікальні повтори** - Скільки унікальних фраз повторюється
- **Частота входжень** - Загальна кількість входжень дублікатів
- **Час (с)** - Час виконання алгоритму
- **Найчастіші фрази** - Топ-10 найчастіших фраз
- **Відсоток повторів (%)** - Відсоток тексту, який є дублікатом

### Інформація про файл

Показує статистику завантаженого файлу:
- **Символів** - Загальна кількість символів
- **Слів** - Кількість слів
- **Унікальних слів** - Кількість унікальних слів
- **Середня довжина слова** - Середня довжина слова в символах
- **Найдовше слово** - Найдовше слово у файлі
- **Найкоротше слово** - Найкоротше слово у файлі

### Підсвітка дублікатів

Текст у нижній частині вікна показує вміст файлу з підсвіткою:
- 🔴 **Червоне** - Дублікати за Rabin-Karp
- 🟢 **Зелене** - Дублікати за Suffix Tree
- 🔵 **Синє** - Семантичні дублікати
- 🟡 **Жовте** - Нечіткий пошук (Fuzzy Match)
- 🟣 **Фіолетове** - N-грам аналіз

### Довжина підрядка

Параметр "Довжина підрядка" контролює, які фрази вважаються дублікатами:
- **Менше** (2-3) - Знаходить більше коротких дублікатів
- **Більше** (10+) - Знаходить тільки довгі дублікати

### Поради

1. **Для швидкого аналізу** - Використовуйте Rabin-Karp
2. **Для точного аналізу** - Використовуйте Suffix Tree
3. **Для пошуку синонімів** - Використовуйте Semantic
4. **Для подібних текстів** - Використовуйте Fuzzy Match
5. **Для фраз** - Використовуйте N-gram Analysis
6. **Для великих файлів** - Збільшіть довжину підрядка
7. **Для малих файлів** - Зменшіть довжину підрядка

## 🎯 Алгоритми

### 1. Рабін-Карп (Rabin-Karp)

**Швидкість:** ⚡⚡⚡⚡⚡ Дуже швидкий
**Точність:** ✓✓✓✓ Висока
**Метод:** Хешування підрядків

Використовує хешування для швидкого пошуку дублікатів. Ідеально для точного пошуку однакових підрядків.

**Кращий для:** Точного пошуку дублікатів

### 2. Суфіксні дерева (Suffix Tree)

**Швидкість:** ⚡⚡⚡⚡ Швидкий
**Точність:** ✓✓✓✓ Висока
**Метод:** Структурний аналіз

Будує суфіксний масив для ефективного пошуку найдовших повторюваних підрядків. Використовує найдовший спільний префікс (LCP).

**Кращий для:** Складних паттернів та найдовших повторень

### 3. Семантичний пошук (Semantic)

**Швидкість:** ⚡⚡⚡ Середній
**Точність:** ✓✓✓⭐ Висока
**Метод:** WordNet синоніми

Знаходить синоніми та семантичні дублікати за допомогою WordNet. Групує слова за їх базовою формою (лемою).

**Кращий для:** Пошуку синонімів та семантичних дублікатів

### 4. Нечіткий пошук (Fuzzy Match) ⭐ НОВИЙ

**Швидкість:** ⚡⚡⚡ Середній
**Точність:** ✓✓✓✓ Висока
**Метод:** Подібність текстів

Знаходить подібні, але не ідентичні текстові патерни. Порівнює рядки та обчислює коефіцієнт подібності.

**Як працює:**
1. Розділяє текст на рядки
2. Порівнює кожен рядок з іншими
3. Обчислює коефіцієнт подібності (0-1)
4. Якщо подібність >= 0.85, позначає як дублікат

**Кращий для:** Документації з варіаціями та опечатками

### 5. N-грам аналіз (N-gram Analysis) ⭐ НОВИЙ

**Швидкість:** ⚡⚡⚡⚡ Швидкий
**Точність:** ✓✓✓✓ Висока
**Метод:** Послідовності слів

Аналізує повторюючи послідовності слів (2-грами, 3-грами, 4-грами). Знаходить повторювані фрази та вирази.

**Як працює:**
1. Токенізує текст на слова
2. Видаляє стоп-слова
3. Створює n-грами (послідовності n слів)
4. Лічить, скільки разів кожен n-грам повторюється
5. Позначає повторювані n-грами як дублікати

**Кращий для:** Пошуку повторюваних фраз та паттернів

### Порівняння алгоритмів

| Алгоритм | Швидкість | Точність | Найкраще для |
|----------|-----------|----------|--------------|
| Rabin-Karp | ⚡⚡⚡⚡⚡ | ✓✓✓✓ | Точні дублікати |
| Suffix Tree | ⚡⚡⚡⚡ | ✓✓✓✓ | Структура |
| Semantic | ⚡⚡⚡ | ✓✓✓⭐ | Синоніми |
| Fuzzy Match | ⚡⚡⚡ | ✓✓✓✓ | Подібності |
| N-gram | ⚡⚡⚡⚡ | ✓✓✓✓ | Фрази |

### UI Архітектура

Додаток використовує **чисту архітектуру** з Premium UI та автоматичним fallback:

```
views/__init__.py
   ↓
   Спробуй завантажити Premium UI
   ↓
   Якщо успішно → Використовуй Premium UI ✅
   ↓
   Якщо помилка → Автоматично використовуй Standard UI ✅
   ↓
main.py
   ↓
   Завжди використовує PremiumLoginView та PremiumMainView
   (fallback відбувається автоматично в __init__.py)
```

**Як це працює:**
1. `views/__init__.py` спочатку намагається імпортувати Premium версії
2. Якщо Premium недоступна (помилка імпорту), автоматично використовує Standard версії
3. `main.py` завжди використовує `PremiumLoginView` та `PremiumMainView`
4. Fallback відбувається прозоро в `__init__.py`

**Переваги:**
- ✅ Чистота - основний код не знає про fallback
- ✅ Надійність - завжди є fallback
- ✅ Простота - мінімум файлів
- ✅ Сумісність - працює з різними версіями бібліотек

**Версії UI:**
- **Premium** - Найкраща версія з customtkinter (основна)
- **Standard** - Базова версія з ttk (автоматичний fallback)

### Додавання нового алгоритму

1. Створіть клас в `models/search_algorithms.py`:
```python
class NewAlgorithm:
    def __init__(self, window_size=10):
        self.window_size = window_size
    
    def find_duplicates(self, text):
        """Повертає (duplicates, positions)"""
        duplicates = {}  # {phrase: [positions]}
        positions = []   # [(phrase, position)]
        return duplicates, positions
```

2. Додайте метод в `models/algorithm_comparator.py`:
```python
def _run_new_algorithm(self, text):
    """Run New Algorithm"""
    start = time.time()
    new_duplicates, new_positions = self.new_algorithm.find_duplicates(text)
    # ... обробка результатів
    return {
        "duplicates": len(new_duplicates),
        "unique_count": len(set(k for k in new_duplicates.keys())),
        "frequency": sum(len(v) for v in new_duplicates.values()),
        "time": time.time() - start,
        "top_phrases": new_top_phrases,
        "positions": new_positions,
        "repeat_percentage": repeat_percentage
    }
```

3. Додайте в `compare_algorithms()`:
```python
results["Новий алгоритм"] = self._run_new_algorithm(text)
```

4. Оновіть UI в `views/main_view.py` для вибору алгоритму.

## 🐛 Проблеми та рішення

### Додаток не запускається
- Перевірте, чи встановлені всі залежності: `pip install -r requirements.txt`
- Перевірте версію Python (потрібна 3.7+)
- Перевірте, чи немає помилок в консолі

### Помилка при завантаженні файлу
- Переконайтеся, що файл є валідним PDF або DOCX
- Перевірте, чи файл не пошкоджений
- Спробуйте інший файл

### Аналіз займає дуже довго
- Спробуйте збільшити довжину підрядка
- Спробуйте менший файл для тестування
- Використовуйте Rabin-Karp замість Suffix Tree

### Немає результатів
- Переконайтеся, що довжина підрядка менша за довжину найкоротшого слова
- Спробуйте інший файл
- Перевірте, чи текст містить дублікати

### Show stats не показує результати
- Переконайтеся, що файл був проаналізований
- Перевірте, чи результати збережені в БД
- Спробуйте видалити файл та проаналізувати його знову

### Помилка NLTK
Якщо отримуєте помилку про відсутність NLTK даних:
```bash
python download_nltk_data.py
```

Або вручну:
